Chapter 11

Streaming Responses & Real-Time Agent Output

TL;DR

  • Streaming cuts perceived latency from 10 seconds to under 1 second. Users see tokens appear immediately instead of waiting for the full response.
  • Day One: SSE (Server-Sent Events) from your FastAPI server to a frontend that renders tokens as they arrive.
  • Production: backpressure handling, mid-stream tool call interception, partial result caching, graceful degradation on stream interruption, and per-token cost tracking.
  • Streaming and tool use interact in a non-obvious way: the model pauses streaming to emit a tool call, your code executes it, and then streaming resumes.
  • Never parse a streaming response as complete JSON until the stream has finished.

Why streaming matters for agents

A non-streaming agent query takes 8-15 seconds. The user sends a message and stares at a spinner while your system retrieves context, calls the LLM, waits for a full response, and then renders everything at once. For a simple question this is tolerable. For a complex multi-step reasoning trace, it feels broken.

Streaming changes the user experience from "waiting for a response" to "watching the agent think." The first token appears in under a second. The quality of the response is identical - only the delivery changes.

Streaming does not make your agent faster. It makes the wait invisible by turning a 10-second pause into a 10-second typewriter effect.

Day One: basic SSE streaming

# server.py - FastAPI streaming endpoint
from fastapi import FastAPI
from fastapi.responses import StreamingResponse
from openai import AsyncOpenAI
import asyncio, json

app = FastAPI()
client = AsyncOpenAI()

async def token_stream(query: str, context: str):
    """Async generator that yields SSE-formatted chunks."""
    stream = await client.chat.completions.create(
        model="gpt-4o",
        messages=[
            {"role":"system","content":"You are a wiki assistant."},
            {"role":"user","content":f"Context:\n{context}\n\nQ: {query}"}
        ],
        stream=True,
        max_tokens=1000,
    )
    async for chunk in stream:
        delta = chunk.choices[0].delta
        if delta.content:
            # SSE format: "data: {json}\n\n"
            yield f"data: {json.dumps({'token': delta.content})}\n\n"
        if chunk.choices[0].finish_reason == "stop":
            yield f"data: {json.dumps({'done': True})}\n\n"

@app.get("/query/stream")
async def stream_query(q: str):
    context = retrieve_context(q)  # blocking retrieval before stream starts
    return StreamingResponse(
        token_stream(q, context),
        media_type="text/event-stream",
        headers={
            "Cache-Control": "no-cache",
            "X-Accel-Buffering": "no",  # disable nginx buffering
        }
    )
// frontend.js - consume the SSE stream
const source = new EventSource('/query/stream?q=How+do+I+restart+payment-svc');
let fullText = '';

source.onmessage = (event) => {
  const data = JSON.parse(event.data);
  if (data.done) {
    source.close();
    return;
  }
  fullText += data.token;
  document.getElementById('response').textContent = fullText;

Production: streaming with tool calls

Tool use introduces a complication: the model cannot stream while it is emitting a tool call. When the LLM decides to call a tool mid-response, it pauses the token stream, emits a complete tool call JSON payload, your code executes the tool, and then the stream resumes with the tool result injected into context. Your streaming handler needs to manage all three states.

# streaming/agent_stream.py
import asyncio, json
from enum import Enum
from openai import AsyncOpenAI

client = AsyncOpenAI()

class StreamEvent(Enum):
    TOKEN       = "token"       # text token to render
    TOOL_START  = "tool_start"  # agent is calling a tool
    TOOL_END    = "tool_end"    # tool call finished, streaming resumes
    ERROR       = "error"
    DONE        = "done"

async def agent_stream(query: str, tools: list, registry):
    """Full agent stream with tool call interception."""
    messages = [
        {"role":"system","content":SYSTEM_PROMPT},
        {"role":"user","content":query}
    ]

    while True:  # loop handles multi-turn tool use
        stream = await client.chat.completions.create(
            model="gpt-4o", messages=messages,
            tools=tools, stream=True, max_tokens=1000,
        )

        accumulated_content = ""
        accumulated_tool_calls = []
        current_tool_call = None

        async for chunk in stream:
            delta = chunk.choices[0].delta
            finish = chunk.choices[0].finish_reason

            # Regular text token
            if delta.content:
                accumulated_content += delta.content
                yield {"event": StreamEvent.TOKEN, "data": delta.content}

            # Tool call started
            if delta.tool_calls:
                for tc_delta in delta.tool_calls:
                    if tc_delta.index == len(accumulated_tool_calls):
                        accumulated_tool_calls.append({
                            "id": tc_delta.id or "",
                            "name": tc_delta.function.name or "",
                            "arguments": "",
                        })
                    accumulated_tool_calls[-1]["arguments"] += (tc_delta.function.arguments or "")

            if finish == "tool_calls":
                # Stream paused for tool execution
                for tc in accumulated_tool_calls:
                    yield {"event": StreamEvent.TOOL_START, "data": {"tool": tc["name"]}}
                    args = json.loads(tc["arguments"])
                    result = await registry.execute(tc["name"], args)
                    yield {"event": StreamEvent.TOOL_END, "data": {"tool": tc["name"], "result": result}}

                    # Inject tool result back into message history
                    messages.append({"role":"assistant","tool_calls":[{
                        "id": tc["id"], "type":"function",
                        "function":{"name":tc["name"],"arguments":tc["arguments"]}
                    }]})
                    messages.append({"role":"tool","tool_call_id":tc["id"],"content":str(result)})
                break  # exit inner loop to start new stream with tool results

            if finish == "stop":
                yield {"event": StreamEvent.DONE, "data": {}}
                return  # fully done

Backpressure: handling slow consumers

# If the client connection is slow, buffer tokens instead of dropping them
async def buffered_stream(agent_gen, max_buffer: int = 50):
    buffer = asyncio.Queue(maxsize=max_buffer)

    async def producer():
        async for event in agent_gen:
            await buffer.put(event)
        await buffer.put(None)  # sentinel

    asyncio.create_task(producer())

    while True:
        event = await asyncio.wait_for(buffer.get(), timeout=30.0)
        if event is None:
            break
        yield event

⚠ Common Mistakes & Gotchas

  1. Buffering the full response before streaming to the client. If your server collects the full LLM response and then streams it to the client, you have eliminated the latency benefit entirely. The client must receive tokens as the LLM generates them, not after.
  2. Not disabling nginx/proxy buffering. Many reverse proxies buffer responses by default. Add X-Accel-Buffering: no to your SSE response headers, or configure your proxy to pass-through streaming responses immediately.
  3. Parsing tool call arguments before the stream finishes. Tool call arguments arrive in fragments across multiple chunks. Concatenate all argument deltas before parsing as JSON - never parse a partial argument string.
  4. No timeout on the tool execution pause. When the stream pauses for a tool call, the user sees a loading state. A hung tool call leaves the user in limbo. Apply the same timeout to tool executions in streaming mode as in non-streaming mode.
  5. Losing the stream on reconnect without partial result caching. If a mobile user loses connectivity mid-stream, they must restart the full query. Cache the assembled tokens with the trace_id so the client can resume from where it disconnected.
